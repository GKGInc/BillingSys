BillingSys — seed test data

This page is deployed with the app (GitHub Pages). It POSTs to the Azure Functions API below — no local server needed. After you push to main, open this same path on your site (see blue box).

URL (after deploy):
…/BillingSys/seed-data.html — bookmark it. (Path matches your repo name + GitHub Pages base href.)
Production API: POST /api/employees and POST /api/projects may require JWT + role (Admin / Manager). Paste a Bearer token if those calls return 401/403. Time entries use POST /api/timeentries. Time entry dates must be ≤ today + 7 days per validator — adjust dates in this file if seed weeks fail.

Clearing test data later: data is in Azure Table Storage. See repo doc docs/AZURE_TABLES_AND_CLEAR_SEED.md for exact table names, Portal steps, and the dev-only API POST /api/admin/clear-seed with body {"confirm":true} (requires Development env or app setting BILLINGSYS_ALLOW_CLEAR_SEED=true).
